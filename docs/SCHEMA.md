# ERC Schema 冻结规则 v1.0.0

## 冻结声明
从 2026-05-08 起，Execution Responsibility Record (ERR) v1.0.0 正式冻结。

## 不可变规则
1. **禁止删除已有字段**。所有已定义的字段永久保留。
2. **禁止修改字段语义**。一个字段的含义在其生命周期内不可改变。
3. **新增字段必须放入 `extensions`**。未来任何新需求，均通过 `extensions` 对象扩展，不得直接向根结构添加必填字段。
4. **向后兼容是最高优先级**。老版本 SDK 必须能够忽略未知的 `extensions` 字段而不出错。

## 版本策略
- **主版本号**：破坏性变更时递增（目前为 1）。
- **次版本号**：在 `extensions` 中新增字段时递增。
- **修订号**：文档或非语义修正时递增。

## 校验
使用 `schemas/err-v1.schema.json` 对任何声称符合 ERC 标准的回执进行校验。

## 法律声明
本 Schema 不构成法律意见。用户需自行评估其在所在司法管辖区的法律效力。
